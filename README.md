# CancerAI.uz

CancerAI — научно-образовательная веб-платформа для анализа цитологических изображений шейки матки. Пользователь загружает изображение, выбирает модель и получает вероятности для классов **HSIL, LSIL, NILM и SCC**.

> Результаты системы не являются медицинским диагнозом и не заменяют консультацию врача.

## Технологии

- Backend: FastAPI и Uvicorn.
- Машинное обучение: PyTorch, timm и TensorFlow/Keras.
- Обработка изображений: Pillow и NumPy.
- Frontend: HTML, CSS, Bootstrap и JavaScript.
- Хранение моделей: локальный каталог `models`, переменная `MODEL_DIR` или Railway Volume `/data/models`.

## Структура проекта

```text
app/
  main.py             # маршруты сайта и REST API
  inference.py        # подготовка изображения и прогноз
  model_registry.py   # регистрация, загрузка и кэширование моделей
pages/                # HTML-страницы
assets/               # CSS, JavaScript, изображения и библиотеки
Dockerfile            # контейнеризация приложения
requirements.txt      # зависимости Python
```

## Запуск локально

```bash
python -m venv .venv

# Windows
.venv\Scripts\activate

# Linux/macOS
source .venv/bin/activate

pip install -r requirements.txt
uvicorn app.main:app --reload --host 127.0.0.1 --port 8000
```

После запуска откройте `http://127.0.0.1:8000`.

## Файлы моделей

В проекте зарегистрированы пять моделей:

1. `cc_mswt_tested.h5`
2. `cc_env2_s_tested.h5`
3. `cc_rf_tested.h5`
4. `cc_ifs_tested.h5`
5. `cc_vit_sts_tested.h5`

Для локального запуска поместите файлы в каталог `models/` в корне проекта. Альтернативный путь можно задать переменной окружения:

```bash
MODEL_DIR=/путь/к/моделям
```

Для Keras-моделей список классов при необходимости задаётся так:

```bash
CLASSES=HSIL,LSIL,NILM,SCC
```

## API

- `GET /api/health` — проверка работоспособности.
- `GET /api/models` — список зарегистрированных моделей.
- `POST /api/predict` — прогноз по загруженному изображению.
- `POST /api/admin/clear-cache` — очистка кэша загруженных моделей.

Интерактивная документация FastAPI доступна по адресу `/docs`.
